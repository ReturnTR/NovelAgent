# 人物构建指南

## 第1步：确定人物职能矩阵

根据剧情需要，将人物分为六大职能，确保不缺失关键角色：

| 职能 | 说明 | 数量建议 | 示例定位 |
|------|------|----------|----------|
| **主角** | 故事核心，承载主线与爽点 | 1-2人 | 概念解析者（特殊系） |
| **核心队友** | 长期陪伴，能力互补，有独立成长线 | 3-5人 | 治愈系辅助、防御系坦克、信息系侦探 |
| **导师/引路人** | 传授知识，推动主角成长，可能牺牲 | 1-2人 | 退休SSS级前辈、神秘组织联络人 |
| **盟友/资源方** | 提供帮助但非长期同行，有自身利益 | 3-6人 | 自由城邦领袖、GSMA内部改革派 |
| **反派/对手** | 制造冲突，分层次递进 | 4-8人 | 技能贵族、隐修会杀手、GSMA鹰派 |
| **氛围/世界观人物** | 展现社会百态，增加真实感 | 不限 | 技能补习班老师、污染区难民、SPD警察 |

## 第2步：设计人物核心标签（关键词）

使用3-5个关键词概括人物本质，便于后续扩展时保持一致性。

**关键词维度建议**：
- **本性维度**：善良、野心、偏执、懦弱、正义感……
- **技能维度**：解析、火焰、治愈、预知……
- **社会维度**：秩序派、自由派、隐修者、底层……
- **成长维度**：天才、努力型、被低估、黑化可能……

## 第3步：构建人物成长弧线

每个重要人物必须有“起始状态 → 变化过程 → 最终状态”的弧线。

| 阶段 | 主角弧线 | 队友弧线示例 | 反派弧线示例 |
|------|---------|-------------|-------------|
| **前期** | 被判定E级，自卑但好奇 | 相信GSMA的优等生 | 认为强者应统治弱者 |
| **中期** | 发现能力真相，开始反抗 | 对GSMA产生怀疑 | 开始不择手段 |
| **后期** | 承担代价，做出选择 | 独立成长，成为支柱 | 被击败或醒悟 |

## 第4步：定义人物关系网络

使用JSON格式记录关系，注意：
- 双向记录（A对B的态度 + B对A的态度）
- 标记关系强度（0-10）和变化趋势（上升/下降/稳定）
- 关键关系要有“关系事件”（何时、何事导致关系变化）

**关系类型库**：
- 血缘：父母、子女、兄弟姐妹
- 情感：恋人、挚友、仇敌、恩人
- 组织：同事、上下级、同门、对手
- 立场：盟友、敌对、利用、中立

## 第5步：填充人物详情（按SQL字段）

逐个字段填写，特别注意：
- **biography**：用时间线列表，只记录“影响人生的大事”
- **cognition**：体现人物的三观，尤其是“对技能的态度”“对秩序的看法”
- **relationship**：必须双向，且随剧情更新
- **race/gender**：非人类种族要详细描述生理特征

## 第6步：进行“人物自洽性测试”

将人物放入关键场景，检查：
- [ ] 该人物的行为是否符合其性格和认知？
- [ ] 该人物与其他人的关系是否一致？
- [ ] 该人物的技能等级和代价是否匹配？
- [ ] 该人物的成长弧线是否有足够的触发事件？

请按上述步骤进行分析，最终将所有涉及到的人物的生成格式为json字符串，顶层格式是列表，表示每一个人物，具体格式如下：

```json
[
    {
        "worldview": "所处的世界观以及时代背景",
        "name": "人物姓名",
        "race": "人物的种族，非人类时需要描述种族特征",
        "gender": "人物的性别，男性，女性，其他种族可能有双性，无性",
        "age": "该人物目前的年龄", 
        "hometown": "人物出生地",
        "biography": ["这个人做了哪些在他的人生中影响比较大的事", "用python列表格式呈现"],
        "personality": "基本概括这个人性格",
        "cognition": "这个人对世界的认知，包括：对世界的理解、对人物的理解等",
        "relationship": {"人物姓名":"关系描述"},
        "keywords": "用3-5个词快速概括人物的本质特征，便于后续检索和归类"
    }
]
```
